## 1. 产品概述

### 1.1 产品背景

随着个人开发者和小型团队越来越多地使用云服务器，对服务器状态的实时监控需求日益增长。当前市场上的监控方案（如 Zabbix、Prometheus + Grafana）功能强大但配置复杂，对于单台服务器的场景显得过于笨重。CloudGuard Monitor 旨在提供一套轻量级、开箱即用的服务器监控方案。

### 1.2 产品愿景

为个人开发者和小型团队提供最简单的云服务器监控体验：一键部署服务端，手机/PC 随时查看服务器状态，异常时实时告警。

### 1.3 目标用户

- **个人开发者：** 拥有 1-3 台云服务器，需要简单监控
- **小型创业团队：** 运维人员有限，需要低成本监控方案
- **独立项目运维人员：** 希望通过手机随时掌握服务器状态

## 2. 功能需求

### 2.1 服务端 Agent

#### 2.1.1 系统指标采集

| 指标类别 | 具体指标 | 采集频率 | 优先级 |
|---------|---------|---------|-------|
| CPU | 使用率、负载均值、每核使用率、进程数 | 5秒 | P0 |
| 内存 | 总量、已用、可用、缓存、Swap 使用率 | 5秒 | P0 |
| 磁盘 | 各分区使用率、IOPS、读写速度 | 30秒 | P0 |
| 网络 | 带宽使用、连接数、包收发率 | 5秒 | P0 |
| 进程 | Top N CPU/内存消耗进程、总进程数 | 10秒 | P1 |
| 系统信息 | 开机时间、OS 版本、内核版本、主机名 | 启动时 | P0 |

#### 2.1.2 服务监控

- **端口存活检测：** 支持 TCP/HTTP/HTTPS 端口检查，可配置检查间隔
- **Docker 容器监控：** 容器状态、资源使用（P1）
- **自定义脚本监控：** 支持执行用户自定义检查脚本（P2）

#### 2.1.3 告警系统

- **阈值告警：** CPU/内存/磁盘超过设定阈值时触发
- **服务告警：** 监控端口不可达时触发
- **告警通道：** App 推送、WebSocket 实时推送
- **告警策略：** 支持告警抑制（防止告警风暴）、告警恢复通知

### 2.2 API 服务

#### 2.2.1 核心 API

| 接口 | 方法 | 描述 | 认证 |
|-----|-----|------|-----|
| `/api/v1/status` | GET | 获取服务器实时状态 | Token |
| `/api/v1/metrics` | GET | 获取历史指标数据 | Token |
| `/api/v1/alerts` | GET | 获取告警列表 | Token |
| `/api/v1/alerts/rules` | POST/PUT | 配置告警规则 | Token |
| `/api/v1/config` | GET/PUT | 服务端配置管理 | Token |
| `/ws/v1/realtime` | WebSocket | 实时数据推送 | Token |

#### 2.2.2 认证机制

采用 API Token 认证。服务端启动时生成随机 Token，通过配置文件或命令行参数传递给客户端。后续版本可扩展支持 JWT 认证。

### 2.3 PC CLI 客户端

- **实时看板：** 终端中展示服务器实时状态的 Dashboard
- **指标查询：** 支持指定时间范围查询历史指标
- **告警管理：** 查看、确认、配置告警规则
- **配置管理：** 远程修改服务端配置
- **输出格式：** 支持 table/json/yaml 多种输出格式

### 2.4 Android App

#### 2.4.1 核心功能

- **仪表盘：** 展示 CPU、内存、磁盘、网络的实时状态
- **历史图表：** 支持查看各指标的历史趋势图（24h/7d/30d）
- **告警中心：** 告警列表、详情、确认操作
- **推送通知：** 告警时发送系统推送通知
- **服务器连接管理：** 配置服务器地址和 Token

#### 2.4.2 设计要求

- Material Design 3 设计规范
- 支持深色/浅色主题
- 最低支持 Android 8.0 (API 26)
- 支持中英文双语

## 3. 非功能需求

### 3.1 性能需求

| 指标 | 目标值 | 备注 |
|-----|-------|-----|
| Agent CPU 占用 | < 2% | 持续运行时均值 |
| Agent 内存占用 | < 50MB | RSS 常驻值 |
| API 响应时间 | < 200ms | P99 延迟 |
| WebSocket 推送延迟 | < 1s | 从数据采集到客户端展示 |
| 数据保留周期 | ≥30天 | 可配置 |
| 并发客户端 | ≥10 | WebSocket 连接 |

### 3.2 安全需求

- 所有 API 通信必须通过 HTTPS/TLS 加密
- Token 存储采用 bcrypt 哈希
- 支持访问 IP 白名单配置
- 敏感操作需记录审计日志

### 3.3 可用性需求

- Agent 崩溃后自动重启（systemd 服务管理）
- 数据本地缓存，网络断开后自动重连
- App 支持离线查看最后一次缓存数据

## 4. 数据设计

### 4.1 数据存储方案

采用 SQLite 作为嵌入式数据库，无需额外数据库服务，符合轻量级定位。采用时序数据压缩策略，古老数据自动降采样。

### 4.2 数据降采样策略

| 时间范围 | 采样粒度 | 存储方式 |
|---------|---------|---------|
| < 24小时 | 原始数据（5秒） | 全量保存 |
| 1-7 天 | 1分钟均值 | 聚合后保存 |
| 7-30 天 | 5分钟均值 | 聚合后保存 |
| > 30 天 | 自动清理 | 可配置保留 |

## 5. 版本规划

### 5.1 MVP (v0.1) — 核心监控

- 服务端：CPU/内存/磁盘/网络基础指标采集

- PC CLI：基础命令行查询工具
- Android App：基础仪表盘

### 5.2 v0.2 — 告警与通知

- WebSocket 实时推送
- 告警规则配置与触发
- App 推送通知
- 历史图表

### 5.3 v0.3 — 增强功能

- 服务/端口监控
- Docker 容器监控
- TLS 加密通信
- CLI 实时 Dashboard
